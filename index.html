친구 매매가 뭐냐면

친구가 사서 물려야 매수 시점입니다. 단톡에서 모두가 "이거 가즈아!" 외칠 때는 이미 늦었어요. 친구가 슬쩍 "그거... 좀 물렸어" 할 때, 그게 진짜 진입 타이밍.

정석 분석은 우리한테 잘 안 먹혀요. 보고서·차트보다 "친구가 물렸다"는 한 마디가 더 정확할 때가 많거든요. LOUIS는 그걸 신호로 바꿔주는 도구입니다.

우리의 원칙

새 기능 만들 때마다 이 셋으로 돌아옵니다.

친구가 물리면 그게 매수
환호 말고 짜증 들릴 때가 진짜 시점. 모두가 살 때 우리는 빠지고, 모두가 후회할 때 우리는 줍습니다. ● ○ ✕ 신호도 그 원리.
치킨값이면 충분
대박 안 노립니다. 안 잃는 게 1순위, 가끔 치킨값 얹히면 그날은 좋은 날. 그게 본업 있는 사람의 현실적인 목표죠.
내 거 보고 너도 만들어
이 LOUIS도 어차피 한 명의 매매 스타일일 뿐이에요. 부동산 보는 친구는 부동산 LOUIS, 코인 하는 친구는 코인 LOUIS 만들면 그만. 같이 가요.

안 하는 것들

원칙만큼 중요한 게 안 하는 것이에요. LOUIS를 LOUIS답게 만드는 목록.

✕ 보장된 수익 약속 — 우린 친구지 자문사가 아니에요
✕ 회원가입·로그인 — 그냥 들어와서 보고 가세요
✕ 푸시 알람 — 본업 방해는 NO
✕ 차트 자랑 — 차트는 도구지 트로피가 아님
✕ 광고 — 본업이 있는데 굳이

5년 뒤에는

친구들이 자기 입맛대로 LOUIS 뜯어고쳐서 쓰고 있으면 좋겠어요. 누구는 부동산만, 누구는 코인만, 누구는 ETF만. 각자 자기 매매 스타일대로.

LOUIS는 한 명의 도구가 아니라 각자의 도구를 만드는 출발점이 됩니다.

2026-04 디자인 정리

"AI가 자동으로 색칠한 느낌"을 빼고 절제된 톤으로 재정리. 색은 진짜 정보를 전달할 때만 사용, 나머지는 무채색.

01
표준 스켈레톤 시스템
데이터 늦게 도착해도 0이나 빈 값으로 보이지 않게. .skel + 사이즈 변형(.skel-text·.skel-badge·.skel-num-lg 등)으로 어디서든 일관된 펄스 애니메이션. 도착 시 .fade-in-soft로 부드럽게 등장. 매수구간/고점근접/과열/즐겨찾기 카드 + RSI/200SMA 인디케이터에 적용됨.
02
성격 태그 무채색 통일
주도주·모멘텀·소형주·회복기대·배당주 — 5색 → 모두 회색(--text-2) + 일반 보더. 텍스트 자체로 구분, 시각적 잡음 제거. 한 종목 행에 색깔이 7가지 등장하던 게 3가지(오렌지/빨강/회색)로 정리.
03
장식 절제
Top Pick 카드 오렌지 그라데이션 → 단색 surface + 좌측 2px 액센트만. 신호 ● 글로우(box-shadow) 제거. 200일선 화살표 빨/파 → 회색. 이모지(💾🔥📅) → 라인 SVG 아이콘. 테마 헤더 앞 컬러 닷 제거.
04
요약 카드 — 매수 구간만 강조
4개 카드의 좌측 색깔 보더 → 매수 구간만 오렌지(LOUIS 정체성: "친구가 물리면 그게 매수"), 나머지(고점·과열·즐겨찾기)는 회색 통일. 라벨도 11px 대문자 모노 + 자간으로 메타 라벨 위계 강화.
05
Top Pick 카드 정보 위계 재정리
"71점 #1 / 200일선 위 · 71점" 점수 중복 → 종목명 우측에 점수 1번만, 우상단 #1은 작은 모노. reason에서 점수 부분 자동 제거. 헤더 "이번 주 주목"도 모노스페이스+자간 → 산세리프+타이트 자간으로 한국어에 맞게.
06
토글 너비 100%
분류·투자 스타일 토글 — .uni-seg.full 적용. 우측 빈 공간 채워 모바일에서 시각적 안정감.
07
사이드바 메뉴 — 라인 아이콘으로
📅🔥📖 이모지 → 라인 SVG (캘린더/막대그래프/노트북). 브랜드 원칙 섹션과 동일한 스펙(viewBox 24, stroke 1.5, round). currentColor 적용으로 한 가지 회색 톤.
08
신호 시각화 — 단추형 ●○✕ → 점수 박스
"구멍 뚫린 단추" 같다는 피드백 반영. 16×16 원 모양 → 30×18 점수 박스(.sg-badge)로 전환. 매수 검토는 오렌지 채움 + 점수(예: 71), 대기는 회색 보더 + 점수, 회피는 빨강 빗금 + ✕. 정보 밀도 향상 — 신호 + 점수 동시 인지. Top Pick 카드도 동일 패턴(작은 사이즈) 적용.
09
테마 분석 — 판도 (주기율표 메타포)
테마 분석 페이지에 새 판도 뷰 추가 (기존 현재/다음 탭 옆). 섹터(행) × V2 점수 강도(열) 매트릭스로 한눈에 시장 전체 구도 파악. 강도 5단계 (HOT 80+ / STRONG 65~80 / NEUTRAL 40~65 / WEAK 20~40 / COLD <20) — LOUIS 토큰(한국식 빨/파, 블룸버그 오렌지)으로 통일. 시즌 진입월/임박 테마는 ★ 마크. 매트릭스 외곽은 오렌지 토글 박스(액센트 컬러)로 클릭 가능 영역 강조. 칩 클릭 시 bottom sheet로 대표 종목 + 강도/등락률/섹터/등급 표시.

2026-04 로직 고도화 — Phase 1 완성

자동매매 시스템 로드맵 Phase 1 (1~3개월)이 한 달 만에 완성되었습니다. 기존 "도달률 + 시즌"만으로 결정하던 신호를 5종 기술 지표 + 시장 국면 기반의 100점 점수 시스템으로 전면 재설계했습니다.

01
데이터 인프라 — 일봉 OHLCV + 시장 지수
종목별 일봉 1년치(252영업일) + KOSPI / QQQ / VIX 지수 자동 수신. localStorage('ohlcv_v1') 캐시, KST 16시 이후 갱신, 백그라운드 동시 4개 처리. 가격 갱신과 분리되어 사용자 첫 화면을 막지 않습니다. 시장 지수는 별도 30분 캐시 (국면은 일중 자주 바뀌지 않음).
02
기술 지표 8종 — 트레이딩뷰 표준 호환
RSI(14) Wilder 스무딩, SMA(200/60/20), EMA, 볼린저 밴드(20일 2σ), ATR(14), MACD(12/26/9 — signal 라인 + histogram + 골든/데드 크로스 검출 완성), 거래량 비율(20일 평균 대비), 표준편차. 모든 함수는 데이터 부족 시 null 반환 — 안전장치. 실제 트레이딩뷰 RSI/SMA 값과 ±2 이내 일치 검증 완료.
03
V2 점수 시스템 — 100점 만점 + 시장 국면 동적 조정
도달률 30 + RSI 20 + MACD 8 + 200SMA 18 + %B 14 + 거래량 10 = 100점 (RSI + MACD = 28점 모멘텀 클러스터). 시즌 보너스 +15/+12/+8.
즉시 회피 (점수 무관): 도달률 95%+ / 등락 -3% 이하 / RSI 70+ AND 도달률 80%+.
4단계 시장 국면(강세장/약상승/불확실/약세장)으로 매수 임계값 자동 조정 — 강세장 +5(신중), 약세장 -10(적극). "친구가 물리면 그게 매수" 정체성과 일치.
OHLCV 부족(<30일) 종목은 자동 V1 폴백.
04
UI 시각화 + Top Pick 안정화
종목 행 — RSI 배지(과매도 오렌지/중립 회색/과매수 빨강) + 200일선 ▲▽ 화살표 + V2 점수 표시.
페이지 헤더 — 시장 국면 배지 (예: ● 강세장 · KOSPI ↑+53.1% · 임계값 +5).
Top Pick — ● 신호 부족 시 ○ 점수 상위로 채워 항상 3개 표시. 가격 갱신마다 카드가 바뀌지 않게 5분 throttle, 사용자 액션(시장/필터/스타일 변경) 시에만 즉시 재정렬.

콘솔 window.__backtest.compare()로 V1 vs V2 vs Buy&Hold 일괄 비교.

08
peak 자동화 — 사용자가 일일이 입력 안 해도 됨
종목 추가 시 peak 가격을 사용자가 직접 알려주는 부담을 제거. OHLCV 1년치 high에서 자동 산출 — 사용자 입력 우선 (이미 peak가 있으면 건드리지 않음), 비어있는 종목만 52주 고점에서 자동 채움. 한국 종목은 100원 단위, 미국 종목은 0.01달러 단위로 라운딩. 캐시 로드 시 즉시 실행 + OHLCV 갱신 후 자동 재실행. 콘솔 window.__autoPeak.run()으로 강제 실행, __autoPeak.status()로 채움 현황 확인.
09
판도탭 — Theme Strength (TS) 산식 분리
판도탭이 V2 매수 점수를 강도로 사용해 강세장에 모두 비어있던 문제 수정. V2는 "지금 살지 말지" (보수적), 판도는 "지금 강한가" (모멘텀)이 필요한데 두 의미가 충돌했음.
새 TS 산식 (0~100점): 20일 등락률 40 + 5일 등락률 25 + 200SMA 거리 20 + RSI 모멘텀 15. V2 점수와 완전히 독립.
임계값: HOT 85+ / STRONG 70~85 / NEUTRAL 50~70 / WEAK 30~50 / COLD <30. 시트에는 TS와 V2 둘 다 표시 (강도와 매수 점수 분리).
검증: 전력기기 6종목 모두 HOT (90+), 약세 종목 NAVER·Tesla는 WEAK·COLD로 정확히 분류.
10
신호 시스템 V1 완전 제거 — 가짜 점수 깜빡임 수정
사용자가 "리노공업 점수가 처음 66 → 33으로 바뀐다"고 보고. 원인: OHLCV 데이터 도착 전 V1로 폴백해 가짜 점수를 보여준 후, 도착 후 V2로 교체되면서 점수가 갑자기 변하는 혼란.
해결: V1 산식 완전 제거. OHLCV 부족 시 "지표 데이터 대기 중" skel 표시 (가짜 점수 안 보여줌). 데이터 안정성 확보 후 정확한 V2 점수 한 번만 표시.
제거 대상: calcSignal 함수 (66줄) · window.__useSignalV2 토글 · window.__signal.compare() V1/V2 비교 도구. renderSignalBadge가 score: null일 때 자동으로 skel 표시.
남은 콘솔 도구: window.__signal.detail(code) · window.__signal.stats(). 백테스트 영역의 V1 룰은 학술적 비교 도구라 그대로 유지.

✦ 다음 단계 (Phase 2, 3~6개월): 포지션 사이징 · ATR 기반 손절익절 · 포트폴리오 리스크 한도.
✦ 현재 콘솔 도구: window.__signal.detail(code) · window.__signal.stats() · window.__backtest.compare() · window.__regime.current() · window.__indicators.test()
⚠ 한계: OHLCV 1년치 한정 · VIX는 미장만(국장은 V-KOSPI 별도 필요) · 슬리피지 미반영 (실거래는 백테스트보다 0.1~0.3%p 낮을 것).

⚠ V2는 신중한 모드입니다. 상승장 끝부분에선 ● 자주 안 나오고, 조정 시작되면 ●가 늘어납니다 ("친구가 물리면 그게 매수" 정체성과 일치). OHLCV 데이터 부족 시 "분석 대기"로 표시됩니다 (가짜 점수 안 보여줌). 콘솔에서 window.__signal.stats()로 전체 점수 분포 확인 가능.

이번 주 주목 (Top Pick)

현재 시장·필터 풀에서 ● 지금 검토 신호 종목 중 점수 상위 3개를 페이지 상단에 표시. 각 카드 우상단에 V2 점수 표시 (예: 82점).

우선순위 = 위 "점수 시스템 V2"의 총점. 시장(국장/미장)·투자 스타일·필터를 바꾸면 Top Pick도 자동 갱신. 카드 클릭 시 해당 종목 행으로 스크롤 + 1.5초 강조.

종목 성격

종목마다 5가지 중 1~2개 부여. 같은 테마 안에서도 성격을 섞어 위험을 분산합니다.

주도주
테마에서 시총·매출 상위. 흐름이 가장 빠르게 반영되어 신호 확인용으로 적합.
모멘텀
시즌·이벤트 직전 가격이 빠르게 움직이는 종목. 진입·청산 타이밍이 핵심.
소형주
시총 작아 상승·하락 모두 큼. 한 종목에 큰 비중을 싣지 않는 것이 원칙.
회복 기대
부진했던 기업이 실적 회복 신호를 보이는 구간. 변동성 크니 실적 발표 전후 주의.
배당주
배당이 높고 가격 변동이 작음. 하락장에서 손실을 줄이는 역할.

목표 도달률 · 상승 여력

현재가와 사전 설정한 목표가(과거 고점 또는 시즌 평균 피크가) 사이 비율로 계산합니다.

목표 도달률=현재가 / 목표가 × 100%
상승 여력=(목표가 - 현재가) / 현재가 × 100%

목표가는 과거 흐름을 보고 정성적으로 정한 값. 백테스트(과거 데이터 검증)와는 별개입니다.

매수·고점 구간 기준

매수 구간목표 도달률 < 70%
고점 근접목표 도달률 70 ~ 95%
과열목표 도달률 ≥ 95%

통계는 5분 이내 갱신된 최신 가격 종목만 포함합니다. 오래된 가격 데이터는 잘못된 신호를 줄 수 있어 제외.

테마 분석 데이터 (자동 산출)

테마 분석 페이지의 모든 지표는 종목 분석 페이지의 실시간 가격에서 자동 산출됩니다 (캘린더의 12개 산업 섹터 기준).

strength=(100 − 평균도달률) × 0.7 + 평균등락률 × 5 + 30
change=테마 종목들의 평균 등락률
leaders=등락률 상위 7개 종목
rank=strength 내림차순 자동 부여
RRG 좌표=(strength, change) 시계열 누적

5분마다 자동 갱신. 시장(국장/미장) 토글 시 즉시 재산출.

최대 보관90일 (초과 시 오래된 데이터 자동 제거)

⚠ 한계: 사용자가 매일 페이지를 열어야 시계열이 누적됩니다. 첫 1~3일은 점만 보이고, 일주일~한 달 후부터 의미 있는 궤적이 형성됩니다.
✦ 서버 추가 시 개선: 맥 스튜디오 서버를 두면 사용자 방문과 무관하게 매일 자동 저장 → 즉시 풀 궤적 가능. 또한 거래대금·시총 등 추가 데이터로 strength 정확도 향상.
